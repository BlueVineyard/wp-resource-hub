# Project Brief: WP Resource Hub

## Overview

WP Resource Hub is a comprehensive WordPress plugin for managing and displaying diverse resource types including videos, PDFs, downloads, external links, and internal content. It provides a complete resource management system with organization, access control, statistics tracking, and flexible display options.

## Core Purpose

Transform WordPress into a robust resource library that can:

- Organize and categorize diverse content types
- Control access to premium or restricted resources
- Track engagement through views and downloads
- Display resources through multiple presentation methods
- Export/import resources for data portability
- Group related resources into collections

## Target Users

- Content creators who need to organize and share multiple resource types
- Educators managing learning materials
- Organizations distributing documentation and downloads
- Content marketers managing asset libraries
- Membership sites with gated content

## Key Requirements

### Resource Types

1. **Video** - YouTube, Vimeo, or local video embeds
2. **PDF** - Document viewer or download
3. **Download** - General file downloads with tracking
4. **External Link** - Links to external resources
5. **Internal Content** - Article-style content with reading time, TOC

### Core Features

- Custom post type with taxonomies (type, topic, audience)
- Access control system (public, logged-in, specific roles)
- Statistics tracking (views, downloads)
- Collection system for grouping related resources
- Import/Export functionality (CSV, JSON)
- Multiple display methods (shortcodes, Gutenberg blocks)
- Template system for theme customization
- Download tracking with protected URLs

## Success Criteria

- Seamless WordPress integration
- Intuitive admin interface
- Fast frontend performance
- Secure file access control
- Accurate statistics tracking
- Extensible architecture for custom resource types

## Technical Constraints

- Minimum PHP 7.4
- Minimum WordPress 5.8
- Must follow WordPress coding standards
- Must use WordPress APIs (no external dependencies)
- Compatible with block editor and classic editor
